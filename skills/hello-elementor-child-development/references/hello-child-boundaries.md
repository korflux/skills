# Hello Elementor Child Boundaries

Use this reference when deciding whether a task belongs in the child theme, parent theme, Elementor addon skills, or generic WordPress skills.

## Hello Child Theme Owns

- Site-specific PHP hooks, filter overrides, CSS overrides, and small additive assets that are local to one site, one install, or one deployment.

## Route To Hello Parent Theme When

- The change should modify the parent package itself or become general parent theme behavior.

## Route To Elementor Addon Skills When

- The task is reusable widget, control, dynamic tag, form action, form field, Pro integration, or editor extension work.
- The task should survive a theme switch.

## Route Outside Hello When

- The task is generic WordPress plugin logic, generic theme work unrelated to Hello, or WordPress Core behavior.
